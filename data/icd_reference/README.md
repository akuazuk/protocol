# Справочники МКБ-10 для сопоставления запросов

## Единственный источник кодов для лексикона и проверки ввода

1. Положите **официальный Excel** МКБ-10 (например из ветки проекта или с сайта Минздрава РБ) в этот каталог, например:
   - `mkb10_minzdrav_rb.xlsx`  
   или укажите путь в переменной окружения **`ICD_RU_XLSX`**.

2. Соберите JSON (формат: массив `{ "code", "title_ru" }` — как в скрипте, колонки A/B, данные с 5-й строки):

```bash
ICD_RU_XLSX=data/icd_reference/ваш_файл.xlsx \
ICD_RU_JSON=data/icd_reference/icd10_ru_mkb10su.json \
python3 scripts/export_icd_ru_from_xlsx.py
```

Совместимая команда (обёртка):

```bash
python3 scripts/export_icd_ru_mkb10su_json.py
```

3. Сервер читает **только** этот JSON. Путь переопределяется переменной **`ICD_RU_JSON`** (по умолчанию `icd10_ru_mkb10su.json`).

Лексическое сопоставление жалобы с МКБ и проверка кодов, введённых пользователем в тексте, выполняются **только по строкам из этого файла**. Старые сторонние полные выгрузки для лексикона не используются.

## Английские названия (опционально)

Файл **`icd10_who_2016_terminal_codes.json`** (WHO 2016) используется **только** как справочник для поля **английского названия** к уже известному коду, не для подбора кодов по тексту жалобы.

Сборка WHO JSON при необходимости:

```bash
python3 scripts/icd10_who_txt_to_json.py
```

## Идея для бэкенда (актуально)

1. Извлекать из текста кандидаты кодов вида `J20.9`, `A00.0` (регулярное выражение в `icd_mkb.py`).
2. Принимать в работу только коды, присутствующие в JSON из Excel.
3. Коды, введённые явно в запросе, используются без лексических гипотез; подбор кодов через Gemini из пула — только при высоком score отбора протоколов (см. `RAG_ICD_GEMINI_*` в `.env.example`).
